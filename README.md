## English

### About

I am a Software Engineer with 3+ years of professional experience across enterprise platforms, ERP, e-commerce and social products. My work sits where product complexity becomes an architecture problem: shaping composable frontends, independently owned services and reliable asynchronous processing. I am also pursuing an MSc in Applied Information Technology at HUIT.

### Engineering Philosophy

I treat architecture as a set of enforceable engineering decisions—not a diagram. A useful boundary has a clear owner, a contract that communicates intent and a change path that does not require unrelated parts of the system to move together. Shared foundations should remove accidental complexity while leaving business ownership local. Reliability should be designed into retry, concurrency and shutdown behavior rather than added after failure.

### Frontend Architecture

I design React and TypeScript platforms with Micro Frontend (MFE) and Feature-Sliced Design. Host composition, modules, features and shared foundations have distinct responsibilities. Metadata-driven routing, navigation, permissions and module registration are paired with generated or shared UI contracts so composition remains deterministic and configuration drift is visible.

**Core expertise:** React · TypeScript · MFE · Feature-Sliced Design · host/module composition · metadata-driven UI · reusable UI foundations

### Backend Architecture

I structure .NET Microservices with Clean Architecture, Domain-Driven Design (DDD) and CQRS. Domain, Application, Infrastructure and API boundaries protect business rules from delivery and persistence concerns. API Gateway policies, typed clients and deliberately scoped contracts integrate services without erasing service ownership.

**Core expertise:** .NET · Microservices · Clean Architecture · DDD · CQRS · API Gateway · typed service clients · contract design

### Distributed Systems

I build RabbitMQ-based workers around idempotency, bounded concurrency, cancellation, graceful shutdown and retry-safe execution. Platform work also includes SQL Server multi-tenancy, reusable libraries, Azure DevOps and CI/CD quality gates for repeatable delivery.

**Core expertise:** RabbitMQ · asynchronous workers · idempotency · multi-tenancy · SQL Server · Azure DevOps · CI/CD

**Broader experience:** Laravel · Angular · Vue.js · gRPC · Elasticsearch · PostgreSQL · MySQL · Docker · OpenAI integration · Dify/RAG · Power BI

### Anonymous Case Studies

#### Enterprise Platform Architecture

- **Problem:** Modular growth was increasing ambiguity around UI composition, service ownership, duplicated configuration and retry behavior.
- **Decisions:** Separate host/module/feature boundaries with MFE and Feature-Sliced Design; establish metadata contracts; align .NET services to Clean Architecture, DDD and CQRS; use typed clients and idempotent workers.
- **Engineering outcomes:** Explicit ownership boundaries, deterministic composition, reusable engineering foundations and retry-safe asynchronous processing.
- **Takeaway:** Architecture scales when boundaries are represented in code, contracts and delivery rules—not only in documentation.

[Read the full case study](https://kaystar645.github.io/portfolio/#/project/enterprise-platform-architecture)

#### Multi-channel Customer Engagement Platform

- **Problem:** Embedded communication, multiple channels and knowledge-assisted responses needed stable integration boundaries and repeatable delivery.
- **Decisions:** Isolate the embeddable widget, channel adapters and knowledge layer; package runtime dependencies consistently; automate release steps through CI/CD.
- **Engineering outcomes:** A reusable widget boundary, replaceable integrations and more consistent environment delivery.
- **Takeaway:** Integration-heavy products stay maintainable when external variability is isolated behind owned adapters.

[Read the full case study](https://kaystar645.github.io/portfolio/#/project/multi-channel-engagement-platform)
